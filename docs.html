Bubble Sort is a basic comparison-based algorithm that repeatedly compares and swaps adjacent elements if they are in the wrong order. After each full pass, the largest unsorted element settles at its correct position at the end of the array‚Äîlike bubbles rising to the top.

    • ‚Üí 8 is in its correct place (sorted).

Pass 2:

  • Compare 3 and 6 ‚Üí No Swap

    • ‚Üí 6 is now correctly placed.

üìò What is Selection Sort?

Selection Sort is a simple comparison-based sorting algorithm. It works by repeatedly selecting the smallest (or largest) element from the unsorted portion and placing it at the correct position in the sorted portion. It‚Äôs like selecting the smallest card from a hand and placing it at the start, one card at a time.

üîÑ How It Works Step by Step

Let‚Äôs say you have this array: [7, 2, 5, 1]

  • Find the smallest in [7, 2, 5, 1] ‚Üí 1
  • Swap 1 with 7 ‚Üí [1, 2, 5, 7]

    • ‚Üí 1 is in its correct position.

Pass 2:

  • Find the smallest in [2, 5, 7] ‚Üí 2
  • No swap needed ‚Üí [1, 2, 5, 7]

    • ‚Üí 2 is now fixed.

Pass 3:

  • Find the smallest in [5, 7] ‚Üí 5
  • No swap needed ‚Üí [1, 2, 5, 7]

    • ‚Üí Now everything is sorted!

üìò What is Insertion Sort?

Insertion Sort is a simple, intuitive sorting algorithm that builds the sorted array one item at a time. It works the same way you might sort playing cards in your hands: by inserting each new card into its correct position relative to the already sorted ones.

üîÑ How It Works Step by Step

Let‚Äôs say you have this array: [4, 3, 6, 1]

  • Start from 2nd element ‚Üí Compare 4 and 3 ‚Üí Swap ‚Üí [3, 4, 6, 1]

    • ‚Üí First two elements are sorted.

  • Compare 6 with 4 ‚Üí No swap ‚Üí [3, 4, 6, 1]

    • ‚Üí First three elements are sorted.

Pass 3:

  • Compare 1 with 6 ‚Üí Swap ‚Üí [3, 4, 1, 6]
  • Compare 1 with 4 ‚Üí Swap ‚Üí [3, 1, 4, 6]
  • Compare 1 with 3 ‚Üí Swap ‚Üí [1, 3, 4, 6]

üìò What is Quick Sort?

Quick Sort is an efficient, comparison-based sorting algorithm that follows the divide-and-conquer approach. It works by selecting a pivot element and then partitioning the array such that:

  • All elements smaller than the pivot are moved to its left
  • All elements greater than the pivot are moved to its right

This process is then applied recursively to the left and right subarrays until the entire array is sorted.

üîÑ How It Works Step by Step (Partition Intuition)

Consider the array: [8, 3, 6, 1, 5]

Step 1: Choose Pivot

  • Select the first element as pivot: 8
  • Rearrange the array so elements < 8 are on the left, > 8 on the right
  • ‚Üí After partition: [3, 6, 1, 5, 8]
  • Now 8 is placed at its correct position

Step 2: Apply Quick Sort on Left Partition

  • Now sort [3, 6, 1, 5]
  • Choose 3 as pivot ‚Üí Partition ‚Üí [1, 3, 6, 5]
  • 3 is placed correctly

Step 3: Continue Recursively

  • Sort [1] (already sorted), then sort [6, 5]
  • Choose 6 as pivot ‚Üí [5, 6]
  • Final array: [1, 3, 5, 6, 8]

üìò What is Merge Sort?

Merge Sort is a stable, divide-and-conquer sorting algorithm. It works by dividing the array into smaller subarrays, sorting them recursively, and then merging the sorted subarrays back together to form the final sorted array.

Merge Sort does not sort the array in place during the divide step‚Äîit creates new arrays during the merge phase. The key operation is merging two already sorted subarrays into a single sorted array.

üîÑ How It Works Step by Step (Divide and Merge Intuition)

Consider the array: [5, 2, 8, 1]

Step 1: Divide

  • Split array into halves ‚Üí [5, 2] and [8, 1]
  • Split again ‚Üí [5], [2], [8], [1]
  • Each element is now a single-element array (base case)

Step 2: Merge Sorted Parts

  • Merge [5] and [2] ‚Üí [2, 5]
  • Merge [8] and [1] ‚Üí [1, 8]

Step 3: Final Merge

  • Merge [2, 5] and [1, 8] ‚Üí [1, 2, 5, 8]
  • ‚úÖ Final sorted array

üìò What is Radix Sort?

Radix Sort is a non-comparison-based sorting algorithm that sorts numbers by processing their digits one place at a time‚Äîfrom the least significant digit (LSD) to the most significant digit (MSD). It is commonly used when sorting integers with a known digit range.

It uses a stable sorting method (like Counting Sort) at each digit level to maintain the relative order of elements. This process continues until all digits are processed, resulting in a fully sorted array.

üîÑ How It Works Step by Step (Digit-by-Digit Intuition)

Consider the array: [170, 45, 75, 90]

Step 1: Process 1's Digit (exp = 1)

  • Compare last digits: [0, 5, 5, 0]
  • Sort based on last digit ‚Üí [170, 90, 45, 75]

Step 2: Process 10's Digit (exp = 10)

  • Digits: [7, 9, 4, 7]
  • Sort based on 10's digit ‚Üí [45, 170, 75, 90]

Step 3: Process 100's Digit (exp = 100)

  • Digits: [0, 1, 0, 0]
  • Sort based on 100's digit ‚Üí [45, 75, 90, 170]

‚úÖ Final sorted array: [45, 75, 90, 170]

üìò What is Counting Sort?

Counting Sort is a non-comparison-based sorting algorithm that works by counting the number of occurrences (frequency) of each unique element. It then reconstructs the sorted array based on these frequencies.

It is most efficient when sorting integers over a small, known range. Counting Sort is not comparison-based, making it faster than comparison sorts like Quick Sort or Merge Sort in specific scenarios.

üîÑ How It Works Step by Step (Frequency-Based Intuition)

Consider the array: [4, 2, 2, 8, 3]

Step 1: Count Frequencies

  • Create a count array of size (max - min + 1)
  • Count occurrences ‚Üí [0, 0, 2, 1, 1, 0, 0, 0, 1]

Step 2: Rebuild Sorted Array

  • Using the count array, place elements in order: [2, 2, 3, 4, 8]
  • Each value is placed based on how many times it appeared

‚úÖ Final sorted array: [2, 2, 3, 4, 8]

The elements inside each bucket are then sorted individually using another algorithm like Insertion Sort. Finally, all the buckets are concatenated to form the final sorted array.

It is most effective when the input is uniformly distributed over a range, such as floating-point numbers or integers with a known spread.
